# Renamed the `no-green-on-arrival-tests` rule to **`no-verified-on-arrival-tests`**, and conformed "green-on-arrival" to "verified-on-arrival" across the DBE rules. The `testing` package's `say-verified-not-green` rule names this compound explicitly ("The compound 'green-on-arrival' likewise becomes 'verified-on-arrival'"), and DBE includes `testing` — so shipping the rule under its old name would have shipped a package violating its own dependency. The **red/green loop** keeps its name: there "green" names the state in a cycle being described (and "red-green-refactor" names the ceremony DBE rejects), not a report that tests pass, which is what the verified-not-green rule governs. Also corrected the rule's stale "four-hinge cycle (actuation → assertion → controls → implementation)" — the cycle is five hinges (naming deferred to the feature's close), per `hinge-cycle`.
